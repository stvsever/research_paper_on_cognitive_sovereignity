\begin{abstract}
\noindent
This paper presents and validates an ontology-driven pipeline for susceptibility auditing under adversarial political persuasion. The design crossed 25 deterministic pseudoprofiles with four attack leaves and ten opinion leaves sampled across four policy domains, yielding 1{,}000 attacked scenarios. The profile ontology retained survey-mappable Big Five, socioeconomic status, political psychology, social context, and demographic features (91 modeled dimensions after exclusion of earlier noise-heavy inventories). The analysis stack combined repeated-outcome structural equation modeling, conditional susceptibility estimation via task-specific ridge models, hierarchical importance decomposition, bootstrap rank uncertainty quantification, and a profile feature network built from the deterministic profile data.

The central finding is methodological: during the present study, opinion-generation stages fell back to deterministic substitute outputs after API-credit exhaustion, producing 100\% fallback use in baseline and post-exposure assessment and zero-valued realism scores. Under those degraded execution conditions, mean adversarial effectivity was 0.10, 49.6\% of rows had positive adversarial effectivity, $\mathrm{ICC}(1)$ for $|\Delta|$ was 0.011, ridge cross-validated $R^2$ was $-0.175$, random-forest out-of-bag $R^2$ was $-0.194$, and the elastic net selected zero features. The robustness layer behaved appropriately: it surfaced weak-signal diagnostics instead of supporting unwarranted moderator claims.

One component remained fully interpretable despite the execution failure: the deterministic profile feature network. The resulting graph contained 91 nodes and 1{,}807 edges at $|\rho|\ge .15$, with density $=.441$, five communities, and modularity $=.313$. Online-behavior, institutional-trust, and Big Five openness/extraversion blocks formed the most connected regions of the feature space, while cross-community bridge diagnostics highlighted variables that link otherwise distinct ontology families. This study demonstrates that the analytical guardrails, visualization layer, and network diagnostics of the platform function correctly under degraded execution, and that interpretable structure is recoverable from deterministic components even when opinion-generation fails.
\end{abstract}

\section{Introduction}

Online influence operations exploit attention, trust, social proof, and identity-congruent framing rather than technical intrusion alone \citep{bennett2018disinformation,lazer2018science,lewandowsky2017posttruth}. In that setting, a useful defensive question is not merely whether manipulative content exists, but whether specific attack mechanisms shift opinion in adversarially favorable directions for some profiles more than for others. Psychographic tailoring has made this question operationally salient because persuasive messages aligned to personality structure can outperform untailored variants \citep{matz2017psychological}. At the same time, resilience interventions increasingly focus on strengthening citizens' ability to recognize and resist manipulation rather than treating misinformation exposure as a purely platform-level problem \citep{kozyreva2020citizens,roozenbeek2022psychological,pennycook2021psychology}.

The present study addresses that defensive problem through an ontology-constrained simulation pipeline. \texttt{PROFILE}, \texttt{ATTACK}, and \texttt{OPINION} are encoded as hierarchical ontologies; each execution instantiates a factorial attack-opinion surface over a deterministic panel of synthetic profiles, elicits baseline and post-exposure opinions, and then estimates directional adversarial effectivity rather than undirected change magnitude. The broader program goal is not to claim that large language model agents are substitutes for human subjects, but to produce a transparent, auditable measurement object that can later be compared against human data using criterion and construct validity tests.

The present study is a deliberate stress test of that analysis stack. It focuses on a set of representative ontology leaves — one attack per mechanism family and a cross-domain opinion sample — and introduces a more advanced conditional susceptibility dashboard and a richer network analysis module aimed at distinguishing dense local hubs from bridge variables in the mixed continuous-plus-dummy profile panel. The paper therefore asks a narrower question than the long-run research program: \emph{does the upgraded analysis pipeline respond defensibly when the execution substrate is degraded, and what interpretable structure can still be recovered from deterministic components of the design?}

\section{Methods}

\subsection{Study Design}

The present study used an attacked-only full factorial design with 25 deterministic pseudoprofiles, four attack vectors, and ten opinion leaves ($25 \times 4 \times 10 = 1{,}000$ scenarios). The four attacks were one leaf from each mechanism family: Misleading Narrative Framing, Astroturf Comment Wave, Fear Appeal Scapegoating Post, and LLM Chatbot Personalized Persuasion. The ten opinion leaves were sampled across defense and national security, foreign policy and geopolitics, immigration and citizenship, and information integrity and platforms.

The active profile panel contained 91 features spanning Big Five traits and facets, socioeconomic standing, employment, institutional trust, ideological positioning, political engagement, online behavior, social capital, age, sex, education, and news-diet/platform indicators. Earlier pilot-only inventories that weakened interpretability and regularized fit stability were excluded from the active feature registry. Profiles were generated deterministically, so the feature matrix and all downstream network results are exactly reproducible from the configuration and random seed.

\subsection{Outcome Definition and Model Stack}

For each attack-opinion-profile triple, the primary outcome was adversarial effectivity:
\begin{equation}
\mathrm{AE}_{iak} = (\mathrm{post}_{iak} - \mathrm{baseline}_{iak}) \times d_k,
\end{equation}
where $d_k \in \{-1,+1\}$ encodes the adversarial direction assigned to opinion leaf $k$. Positive values indicate movement in the adversarially intended direction; negative values indicate resistance or backfire.

The analysis stack comprised five layers. First, profile-level variance partitioning used $\mathrm{ICC}(1)$ to estimate the share of variance attributable to stable between-profile differences \citep{shrout1979intraclass}. Second, a repeated-outcome SEM regressed attack-averaged effectivity indicators on profile moderators, treated as a small-panel descriptive diagnostic rather than a confirmatory latent model \citep{rosseel2012lavaan}. Third, task-specific ridge models were fit for each attack-opinion cell, then aggregated into a conditional susceptibility score using reliability weights derived from sample size and cross-validated error. Fourth, bootstrap resampling at the profile level quantified rank instability and coefficient uncertainty; false-discovery-rate control used the Benjamini-Hochberg procedure \citep{benjamini1995controlling}. Fifth, a profile feature network was computed from the deterministic profile matrix using Spearman correlations with threshold $|\rho|\ge .15$.

\subsection{Advanced Profile Feature Network}

The network analysis is designed for a mixed hierarchical panel in which continuous trait scales coexist with one-hot categorical indicators. Edge weights use $|\rho|$, while path-based metrics use distance $= 1/|\rho|$. Alongside degree, eigenvector centrality, PageRank, closeness, and betweenness, the network export includes weighted degree (strength), participation coefficient, bridge ratio, within-module $z$ score, sign balance, positive-versus-negative correlation strength, community size, and $k$-core index. In the interactive explorer, community convex hulls demarcate detected clusters, force-directed layout enables manual spreading of dense regions, and lasso multi-selection supports cohort-level metric comparison. This distinction matters methodologically: in a dense mixed panel, high degree alone does not indicate whether a variable is a local hub within one ontology block or a bridge that ties otherwise separated families together.

\subsection{Execution-Integrity Diagnostics}

Execution quality was audited explicitly. Baseline and post-exposure stages record fallback usage, realism and plausibility review outcomes, and downstream robustness summaries. This is central to the inferential posture of the platform: if execution fails, the analysis should reveal that failure and downgrade claims instead of allowing visually polished but methodologically indefensible conclusions to pass unchallenged. That design principle is especially important in adversarial-simulation work, where reviewers are rightly skeptical of overfit dashboards and overly certain behavioral narratives.

\section{Results}

\subsection{Execution Integrity Dominates Interpretation}

The present study completed structurally but failed substantively at the opinion-generation layer. Baseline and post-exposure assessment used deterministic fallback outputs for every row, and attack-review scores remained at zero throughout the realism pipeline. Those diagnostics alone imply that the present study cannot support substantive claims about persuasion dynamics. The appropriate interpretation is therefore constrained from the start: this is a validation of the upgraded analysis stack under degraded execution, not a behavioral finding about real or human-like susceptibility.

The descriptive outcomes are consistent with that interpretation. Mean absolute displacement was 34.23 points, mean adversarial effectivity was 0.10, and 49.6\% of rows showed positive adversarial effectivity. The pipeline produced non-trivial movement magnitudes but essentially random directional success. Had the pipeline reported strong profile moderators under those conditions, that would have been a methodological red flag.

\subsection{Predictive Diagnostics Correctly Collapse}

Variance partitioning and out-of-sample diagnostics all converged on the same conclusion. $\mathrm{ICC}(1)$ for absolute shift was 0.011, indicating that almost all variance in $|\Delta|$ sat within profiles across tasks rather than between profiles. Ridge cross-validated $R^2$ was $-0.175$, random-forest out-of-bag $R^2$ was $-0.194$, and the elastic net selected zero features. Together these results show that the upgraded moderation stack is behaving defensibly: under fallback-generated opinion data, it refuses to manufacture predictive structure.

This negative result is useful. It demonstrates that the methodological guardrails do what they are supposed to do under adverse conditions. A less defensible system might still return a visually compelling ranking, a handful of unstable coefficients, and an overconfident narrative. Here, the diagnostics instead converge on the absence of reliable profile-level signal.

\subsection{Network Structure Remains Interpretable}

Unlike the opinion-generation stages, the profile feature network is unaffected by API failure because it is computed from the deterministic profile matrix. The graph contained 91 nodes and 1{,}807 edges at $|\rho|\ge .15$, with density $=.441$, five communities, and modularity $=.313$. That topology indicates a moderately clustered but still highly cross-connected feature space. The most connected regions concentrated in online-behavior indicators, institutional-trust variables, and Big Five openness/extraversion blocks, which is consistent with the way the deterministic profile generator reuses coherent latent structure across related features.

The more informative result is not simply that the graph is dense, but that bridge diagnostics separate local hubs from cross-family connectors. Variables with high participation or bridge ratio are not merely central; they tie otherwise distinct ontology families together. In practice, these are the nodes worth scrutinizing when reviewing redundancy, multicollinearity, or hidden coupling between dummy-coded demographics and continuous psychographic scales. This is exactly the kind of diagnostic that a reviewer would ask for in a mixed hierarchical feature panel.

\subsection{Why the Negative Result Is Analytically Important}

Peer review should be harsher on synthetic susceptibility studies than on ordinary exploratory dashboards, not more permissive. The present study therefore passes only if the upgraded system remains conservative when the data-generation chain degrades. On that criterion, the study is informative: the inferential layers that depended on modelled opinion responses degraded toward noise, while the deterministic layer that depended only on feature construction remained stable and interpretable. That separation between \emph{execution-sensitive} and \emph{execution-invariant} components is itself a useful validation outcome.

\section{Discussion}

The main contribution of the present study is that it demonstrates analytical self-restraint. The dashboard, tables, and manuscript no longer treat every completed execution as if it merited substantive inference. Instead, the platform now exposes execution quality, shows when predictive fit collapses, and routes attention toward the parts of the pipeline that remain interpretable. That is the correct methodological stance for an ontology-driven simulation framework that will eventually be compared against real opinion data.

Three reviewer-facing points follow directly. First, this study does \emph{not} justify any claim that certain personalities or demographic groups are more susceptible to adversarial persuasion. The fallback-dominated opinion stages invalidate that inference. Second, the network analysis should not be mistaken for evidence about persuasion outcomes. It characterizes the geometry of the modeled feature panel, not the success of attacks. Third, the upgraded network explorer is substantively useful because it diagnoses redundancy, bridge structure, and mixed-feature coupling before any higher-stakes modeling claims are made.

This perspective is also operationally practical. A replication with funded API access from stage 02 onward can now reuse the upgraded stack without further architectural changes. If that replication still yields near-zero predictive fit, the correct conclusion will be that the broader multi-domain attack surface contains weak profile-level signal. If instead real opinion-generation outputs recover stable task structure or bridge-sensitive moderators, the upgraded diagnostics and dashboard are already in place to show that signal defensibly.

\section{Conclusion}

The present study should be read as a validation of analytical infrastructure, not of substantive persuasion findings. The study confirms that the upgraded analysis stack can:
\begin{enumerate}
  \item preserve a reproducible ontology-driven factorial design over profiles, attacks, and opinions;
  \item surface execution failure rather than masking it with overconfident moderator claims;
  \item quantify predictive collapse through ICC, ridge, random-forest, elastic-net, and bootstrap diagnostics; and
  \item recover interpretable deterministic structure through an advanced profile feature network with hub-versus-bridge diagnostics and an interactive explorer that supports force-directed layout, community hull visualization, lasso multi-selection, and shortest-path analysis.
\end{enumerate}

That is a stronger result than a noisy list of nominally significant coefficients would have been. The next scientifically meaningful step is straightforward: restore API access, re-execute stages 02–09 for the same design, and then evaluate whether the upgraded diagnostics support any profile-level susceptibility claims under normal execution.

\section*{Dual-Use Ethics Statement}

This project models mechanisms that could, in principle, be repurposed for adversarial targeting. The intended use is defensive: auditing vulnerability structure, stress-testing analytical claims, and identifying where resilience interventions may be most needed. No real individuals are modeled; all profiles are synthetic; and the present study yields no operationally actionable targeting signal. Any future deployment on real data would require ethics review, access controls, and explicit prohibition of offensive use.

\section*{Reproducibility Statement}

The full simulation pipeline is implemented in Python and driven by hierarchical JSON ontologies. All analyses are reproducible from the ontology files and the random seed (seed = 100, bootstrap samples = 200). Stage-level artifacts — delta summaries, ICC outputs, OLS coefficient tables, SEM fit indices, bootstrap CSVs, ridge reliability weights, and hierarchical importance breakdowns — are stored under the run's \texttt{stage\_outputs/} directory with per-stage \texttt{manifest.json} files for hash-level verification. The deterministic profile feature network is reproducible independent of API availability. The opinion-generation components used fallback outputs in the present study; to obtain real opinion data, restore API credits and re-execute from stage 02.
